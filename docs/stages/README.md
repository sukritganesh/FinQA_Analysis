# Stage Notes

This folder is a lightweight working notebook for the project.

Each file corresponds to one stage of the FinQA pipeline so we can:

- write down design decisions before coding
- track what is done vs. still open
- record tradeoffs and assumptions
- keep implementation aligned with the take-home story

These notes are meant to evolve as the project evolves.

Suggested workflow:

1. Write the goal of the stage.
2. List the exact outputs we want from that stage.
3. Record open questions before coding.
4. Implement the stage.
5. Come back and update what changed, what worked, and what still needs work.
