Program format:
Use a linear execution trace. Later function calls may reference previous results as #0, #1, #2, etc.
Example chain: subtract(153.7, 139.9), divide(#0, 139.9)
In that example, #0 means the result of subtract(153.7, 139.9).
Do not put one operation inside another operation. Every operation must appear as its own step in the output line.
All operations have exactly two arguments. They use the following format:
operation(argument1, argument2)

Below is a list of allowed operations.
ONLY use the operations listed below. DO NOT use any other operations.
Make sure only two arguments are present!

Allowed operations:
add(a, b)
subtract(a, b)
multiply(a, b)
divide(a, b)
greater(a, b)
table_sum(row name, none)
table_average(row name, none)
table_max(row name, none)
table_min(row name, none)

Table operations:
Use table_sum/table_average/table_max/table_min when the question asks for an aggregation across values in one rendered table row.
The first argument should be the row name to aggregate.
Use none as the second argument.
Do not use table operations to retrieve a single cell value.
If the evidence gives a single needed cell value, copy the number directly into add/subtract/multiply/divide/greater.

Examples:
Net change: subtract(445, 333)
Single-cell table values: subtract(455, 333), not subtract(table_sum(net profit 2003, none), table_sum(net profit 2004, none))
Percentage change from 23.3 to 40.3: subtract(40.3, 23.3), divide(#0, 23.3)
Ratio or portion: divide(151.6, 3734.3)
Sum: add(20.8, 1.4)
Product followed by unit conversion: multiply(736, 21.13), multiply(#0, 1000)
Yes/no comparison: multiply(554, 22.4), multiply(#0, 6000), multiply(2.4, 500000), greater(#1, #2)
Direct yes/no answer if no computation is needed: yes
Nested math rewritten as linear steps:
Incorrect: divide(subtract(193.5, 100.0), 100.0)
Correct: subtract(193.5, 100.0), divide(#0, 100.0)
Table row aggregation examples (only use if a table row has MORE THAN one value):
Table sum across one row: table_sum(profit in 2014, none)
Table average across one row: table_average(purchases in 2018, none)
Table maximum across one row: table_max(foreign currency reserves, none)
Table minimum/range across one row: table_min(expected volatility, none), table_max(expected volatility, none), subtract(#1, #0)

Some things to worry about:

When calculating a percentage difference between two numbers, make sure you do not switch the order of the numbers.
Make sure to not use table_sum if the table row contains only one element.
If the output program has multiple functions, make sure that these are separated by a comma followed by a space.

DO NOT include more than 2 arguments in an operation.
Incorrect: add(1, 2, 3)
Correct: add(1, 2), add(#0, 3)

DO NOT include nested operations. Use a sequence of operations, with the results of previous operations indexed with #0, #1, #2, etc.
Incorrect: divide(subtract(193.5, 100.0), 100.0)
Correct: subtract(193.5, 100.0), divide(#0, 100.0)
Incorrect: subtract(add(1, 2), add(3, 4))
Correct: add(1, 2), add(3, 4), subtract(#0, #1)

DO NOT include operands inside the output. You MUST use the allowed operations.
Incorrect: add(30 - 20, 10)
Correct: subtract(30, 20), add(#0, 10)
