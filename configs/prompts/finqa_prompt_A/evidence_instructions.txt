How to read evidence:
Evidence ids beginning with [text_] are sentences or text snippets from the financial report.
Evidence ids beginning with [table_] are rendered table rows.

Table evidence follows this schema:
<stub column name> the <row name> of <column name> is <cell value> ; the <row name> of <next column name> is <next cell value> ;

The stub column name is the first table header cell when it exists. Sometimes, the stub column name is not present.
The row name is the first cell of the table row.
The column name is the header for the cell value.
Each "the <row name> of <column name> is <cell value> ;" phrase describes one table cell.
A rendered table row may contain multiple cells separated by semicolons; these cells will all have the same row name but different column names.
If a table row contains only a single cell (no semicolons to separate), DO NOT use any table functions such as table_sum, table_min, or table_max.

Example text evidence:
[text_9] net revenue utility following is an analysis of the change in net revenue comparing 2015 to 2014 .

This text evidence says the relevant table compares 2015 net revenue to 2014 net revenue.

Example table evidence:
[table_6] ( square feet in millions ) the leased facilities2 of unitedstates is 2.1 ; the leased facilities2 of othercountries is 6.0 ; the leased facilities2 of total is 8.1 ;

In that table evidence:
stub column name = ( square feet in millions )
row name = leased facilities2
column names = unitedstates, othercountries, total
cell values = 2.1, 6.0, 8.1

For that example, finding the proportion of square feet for leased facilities2 that is in unitedstates:
divide(2.1, 8.1)
